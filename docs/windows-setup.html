Windows setup

A little one-time preparation gets your Windows computer ready to install jstats while it’s distributed from GitHub. It takes a few minutes, and once jstats reaches CRAN none of it will be needed.

Using any R package is really two steps that are easy to blur together. Installing downloads the package onto your computer — you do this once. Loading switches it on for your current session — you do this each time you start R, with library(). (Think of installing as buying a book for your shelf, and loading as taking it down to read.)

jstats is currently distributed through GitHub — a website where software is shared and developed — rather than from CRAN, the central archive you downloaded R from. That difference is the only reason your computer needs a little one-time preparation now, and why the install command on the next page looks slightly different from an ordinary package install. Once jstats reaches CRAN, that preparation falls away and installing becomes a single ready-made download. Until then, this page covers the small bit your computer needs.

Install Rtools (recommended)

Rtools is a small set of build tools for Windows. jstats itself doesn’t need it — jstats is written entirely in R, so it installs and runs without compiling anything. But there’s a practical reason to install it now anyway. While jstats installs from GitHub, RStudio shows a warning every time you install or update a package if it can’t find Rtools on your computer. The warning begins “Rtools is required to build R packages but is not currently installed” and looks alarming — but it is harmless here: nothing is actually wrong, and the install still works. The catch is that it reappears on every update, and jstats updates often during this phase, so the simplest way never to be bothered by it is to install Rtools once, now.

To install it:

  1. Go to https://cran.r-project.org/bin/windows/Rtools/.
  2. Choose the version that matches the version of R you installed — for example Rtools45 for R 4.5.x, or Rtools44 for R 4.4.x. (The Rtools page explains how to tell which you need. Matching matters — an unmatched version won’t clear the warning.)
  3. Run the installer with its default settings, then restart RStudio (close it and reopen it) so it picks up the new tools.

After this, the Rtools warning won’t appear when you install or update jstats.

NoteWhat the warning looks like (if you skip Rtools)

If you’d rather not install Rtools, you don’t strictly have to — jstats works perfectly without it. You’ll just see this warning each time you install or update jstats, beginning “Rtools is required to build R packages but is not currently installed”:
