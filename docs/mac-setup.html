Mac prerequisites

A short note on preparing a Mac to install jstats from GitHub. The honest summary is that most Mac users need nothing on this page — jstats installs and runs cleanly as it is. It’s here as a safety net for the rare hitch, and to answer a question Mac users often arrive with (XQuartz). Once jstats reaches CRAN, none of it applies.

Using any R package is really two steps that are easy to blur together. Installing downloads the package onto your computer — you do this once. Loading switches it on for your current session — you do this each time you start R, with library(). (Think of installing as buying a book for your shelf, and loading as taking it down to read.)

jstats is currently distributed through GitHub — a website where software is shared and developed — rather than from CRAN, the central archive you downloaded R from. That difference is the only reason your computer needs a little one-time preparation now, and why the install command on the next page looks slightly different from an ordinary package install. Once jstats reaches CRAN, that preparation falls away and installing becomes a single ready-made download. Until then, this page covers the small bit your computer needs.

You probably need nothing extra

jstats is written entirely in R, so installing it doesn’t compile anything, and the install command on the next page (upgrade = "never") tells R not to rebuild jstats’s supporting packages either. On a normal Mac setup, that means jstats installs with no build tools, no compiler, and no warning. So the two sections below are only for the uncommon case where something goes wrong — you can carry on to Step 5 — Install jstats now and come back only if you hit one of these.

If an install ever asks to compile (build tools)

Very occasionally an install may try to build a supporting package from source and report a build or compilation error. If that happens, work through these in order — most people never get past the first step:

  1. Answer “no” to the compile prompt. If R asks whether to “install from sources the package which needs compilation,” type n and press Enter, then run the install again — R will fetch a ready-made version instead. This resolves almost every case.
  2. If it still fails, open the Terminal app (macOS’s built-in command-line window — it’s in the Applications folder, under Utilities) and run xcode-select --install to add Apple’s command line tools, then restart R and retry.
  3. If the error specifically mentions gfortran or a Fortran compiler, the community-maintained macrtools package installs the full matching toolchain for you — see https://mac.thecoatlessprofessor.com/macrtools/.

About XQuartz

You may have heard that R on a Mac “needs XQuartz.” For jstats, it does not.

XQuartz is the X11 windowing system for macOS (Apple stopped bundling X11 years ago). Some R packages rely on it — notably anything built on Tcl/Tk (such as R Commander), the rgl package for 3-D graphics, and R’s older X11 graphics device. jstats uses none of these: its graphics go through the Mac’s native quartz device, so it never touches X11. If you only use jstats, you can ignore XQuartz entirely.

If you later install a different package that does need it, R tells you plainly when you try to use that feature — you won’t be left guessing. At that point, install XQuartz from https://www.xquartz.org/ and then restart your Mac (XQuartz loads at login).
